Data Loading and Cleaning

data_clean <- function(data_week2) {
  #remove NA values
  data_week2 <- data_week2 %>%
    na.omit()
  
  #remove negative fare amount
  data_week2 <- data_week2 %>%
    filter(fare_amount >= 0)
  
  data_week2 <- data_week2 %>%
    filter(passenger_count != 0)
  
  data_week2 <- data_week2 %>%
    filter(trip_distance != 0)
  
  #Data Type Conversion (Convert datetime columns)
  data_week2$tpep_pickup_datetime <- as.POSIXct(data_week2$tpep_pickup_datetime, format = "%Y-%m-%dT%H:%M:%SZ")
  data_week2$tpep_dropoff_datetime <- as.POSIXct(data_week2$tpep_dropoff_datetime, format = "%Y-%m-%dT%H:%M:%SZ")
  
  #Handling RatecodeID (Remove rows with invalid values)
  data_week2 <- data_week2 %>%
    filter(RatecodeID >= 1 & RatecodeID <= 6)
  
  #remove invalid payment types
  data_week2 <- data_week2 %>%
    filter(payment_type >= 1 & payment_type <= 4)
  
  #Remove Inconsistent Data (Remove rows with tpep_dropoff_datetime earlier than tpep_pickup_datetime)
  data_week2 <- data_week2 %>%
    filter(tpep_dropoff_datetime >= tpep_pickup_datetime)
  
  #Cleaning Geographical Coordinates (Assuming valid range for latitude and longitude)
  data_week2 <- data_week2 %>%
    filter(
      pickup_longitude >= -180 & pickup_longitude <= 180,
      pickup_latitude >= -90 & pickup_latitude <= 90,
      dropoff_longitude >= -180 & dropoff_longitude <= 180,
      dropoff_latitude >= -90 & dropoff_latitude <= 90
    )
  
  #categorize variables
  data_week2$VendorID <- as.factor(data_week2$VendorID)
  data_week2$payment_type <- as.factor(data_week2$payment_type)
  data_week2$store_and_fwd_flag <- as.factor(data_week2$store_and_fwd_flag)
  data_week2$RatecodeID <-as.factor(data_week2$RatecodeID)
  
  data_week2 <- data_week2 %>%
    mutate(
      pickup_hour = hour(tpep_pickup_datetime),
      pickup_time_category = case_when(
        pickup_hour >= 5 & pickup_hour < 12 ~ "Morning",
        pickup_hour >= 12 & pickup_hour < 17 ~ "Afternoon",
        pickup_hour >= 17 & pickup_hour < 21 ~ "Evening",
        TRUE ~ "Night"
      ),
      dropoff_hour = hour(tpep_dropoff_datetime),
      dropoff_time_category = case_when(
        dropoff_hour >= 5 & dropoff_hour < 12 ~ "Morning",
        dropoff_hour >= 12 & dropoff_hour < 17 ~ "Afternoon",
        dropoff_hour >= 17 & dropoff_hour < 21 ~ "Evening",
        TRUE ~ "Night"
      )
    ) %>%
    select(-pickup_hour, -dropoff_hour) 
  
  
  data_week2 <- data_week2 %>%
    mutate(day_of_week = weekdays(tpep_pickup_datetime))
  
  lat_bins <- c(-Inf, 40.65, 40.75, 40.80, 40.85, 40.7128, Inf)
  lon_bins <- c(-Inf, -74.03, -73.95, -73.90, -73.80, -74.0060, Inf)
  
  # Define labels for boroughs
  location_labels <- c("Outside NYC", "Bronx", "Manhattan", "Queens", "Brooklyn", "Staten Island")
  
  # Assign borough labels based on bins for pickup location
  data_week2$pickup_location <- cut(data_week2$pickup_latitude, breaks = lat_bins, labels = location_labels, include.lowest = TRUE)
  
  # Assign borough labels based on bins for dropoff location
  data_week2$dropoff_location <- cut(data_week2$dropoff_latitude, breaks = lat_bins, labels = location_labels, include.lowest = TRUE)
  
  data_week2 <- data_week2 %>%
    select(-dropoff_longitude, -dropoff_latitude, -pickup_longitude, -pickup_latitude, -total_amount, -tpep_pickup_datetime, -tpep_dropoff_datetime)
  
  return(data_week2)
  }

Summary Report

Data Cleaning - The cleaning process involved handling missing values, removing negative fare amounts, and filtering out inconsistent or invalid data.Geographical coordinates were cleaned to ensure they fall within valid ranges.Categorical variables were appropriately converted, and new features like time categories and day of the week were created.

Exploratory Data Analysis (EDA) - EDA included visualizations to understand relationships between variables.Scatter plots, box plots, histograms, and bar plots provided insights into the distribution and relationships in the data.Correlation analysis revealed associations between numerical variables.

Sampling - Random samples were taken from the cleaned datasets for training and testing purposes. The sample sizes were 10% of the cleaned data for both training and testing.

Modelling - A linear regression model was constructed to predict tip amounts based on various features. The model included predictors such as trip distance, payment type, day of the week, etc. The model performance was evaluated using the mean squared error (MSE) on the testing data.

Model Interpretation - The intercept of 1.836 represents the estimated tip amount when all other predictors are zero.The coefficients provide insights into the impact of each predictor on the predicted tip amount.For example, an additional passenger is associated with a decrease of 0.0058 units in the predicted tip amount.

Model Performance - The Mean Squared Error (MSE) was calculated on the testing data to assess model performance which here is 2.565. Residual plots were examined to check for model assumptions and identify potential outliers.The residuals vs fitted plot showed some outliers, indicating areas where the model could be improved. Normal Q-Q plot and Scale-location plot provided insights into the distribution and variance of residuals. Residuals vs Leverage plot highlighted potential influential data points.

The model provides a starting point for predicting tip amounts based on the given features. Model performance, as indicated by the MSE, gives a measure of the prediction accuracy. Areas for improvement include handling outliers and exploring more advanced modeling techniques like regularization (e.g., Lasso) to enhance predictive accuracy and reduce the number of variables.
